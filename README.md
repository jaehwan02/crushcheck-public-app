# CrushCheck Public Demo

이 폴더는 원본 앱을 그대로 공개하지 않기 위해 만든 공개용 Expo 데모 앱입니다. 실제 인증, 결제, 푸시, 운영 API, 서버 주소, 토큰 갱신, 학교 검증 로직은 포함하지 않았고, 평가 가능한 핵심 흐름만 데모 API로 재작성했습니다.

## 실행

```bash
cd crushcheck-public-app
npm install
npm run start
```

웹으로 확인하려면 다음 명령을 사용할 수 있습니다.

```bash
npm run web
```

타입 안정성은 다음 명령으로 확인합니다.

```bash
npm run typecheck
```

## 공개용 핵심 기능

- 홈에서 받은 마음과 보낸 마음 목록 조회
- 친구 검색 후 마음 메시지 전송
- 전송 완료 화면에서 상세 화면 이동
- 상세 화면에서 메시지, 힌트, 상태, 날짜 확인
- 설정 화면에서 정상, 빈 응답, 오류 모드를 바꿔 예외 처리 확인

## 폴더 구조

```text
app/            Expo Router 화면과 탭 라우팅
api/            실제 서버 대신 사용하는 데모 API 레이어
components/     공통 UI 컴포넌트
constants/      색상, 간격, 터치 크기 등 디자인 상수
data/           공개 가능한 시드 데이터
hooks/          화면별 데이터 로딩 훅
store/          Zustand 전역 상태
types/          데이터 타입 정의
docs/           평가 설명용 문서
```

## 평가 흐름

1. 홈 화면에서 초기 데모 피드와 토큰 잔액을 확인합니다.
2. 보내기 탭에서 친구를 검색하고 3자 이상의 메시지를 입력합니다.
3. 전송 완료 화면에서 상세 화면으로 이동해 생성된 신호를 확인합니다.
4. 설정 탭에서 빈 응답과 오류 모드를 바꿔 Empty/Error State를 확인합니다.
5. 데모 데이터 초기화로 정상 모드와 초기 시드 데이터를 복원합니다.

세부 점검 항목은 `docs/QA_CHECKLIST.md`에 정리했습니다.

## 공개하지 않은 항목

- 실제 백엔드 API URL과 인증 헤더 처리
- Apple/Google 로그인과 토큰 갱신
- 학교 인증, 푸시 알림, 결제, 운영 배포 설정
- 매칭 추천이나 운영 정책처럼 제품 내부 판단이 들어간 로직

이 데모는 위 항목을 구현한 척하지 않습니다. 공개 가능한 코드로 기능 흐름과 설계 이해도를 설명하기 위한 별도 버전입니다.
