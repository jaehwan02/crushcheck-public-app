# QA Checklist

공개 데모를 제출하거나 시연하기 전에 아래 흐름을 확인합니다.

## 실행 전 확인

- `npm install`이 성공합니다.
- `npm run typecheck`가 통과합니다.
- `npm run web` 또는 Expo Go 실행 화면이 열립니다.

## 홈 화면

- 초기 진입 시 받은 마음 2개, 보낸 마음 1개가 표시됩니다.
- 새로고침 버튼을 눌러도 목록과 토큰 잔액이 유지됩니다.
- 받은 마음 또는 보낸 마음 카드를 누르면 상세 화면으로 이동합니다.

## 보내기 화면

- 친구 검색어를 입력하면 이름 또는 아이디 기준으로 목록이 필터링됩니다.
- 친구를 선택하지 않으면 전송 버튼이 비활성화됩니다.
- 3자 미만 메시지는 검증 오류를 표시합니다.
- 80자까지 입력할 수 있고 글자 수 카운터가 갱신됩니다.
- 정상 전송 후 완료 화면과 상세 화면 이동이 동작합니다.
- 빠른 메시지 템플릿을 눌러 입력창이 채워집니다.
- 메시지를 지우면 버튼이 비활성화되고, 유효성 에러 텍스트가 즉시 사라집니다.

## 신호 상세 화면

- 신호 상태와 함께 만료까지 남은 기간이 보여야 합니다.
- 만료 상태일 때 안내색(빨간 계열)이 적용되어야 합니다.

## 설정 화면

- 정상, 빈 응답, 오류 모드 설명이 각각 표시됩니다.
- 빈 응답 모드에서는 홈과 친구 목록의 Empty State가 보입니다.
- 오류 모드에서는 Error State와 다시 시도 버튼이 보입니다.
- 데모 데이터 초기화 후 정상 모드와 초기 데이터가 복원됩니다.
